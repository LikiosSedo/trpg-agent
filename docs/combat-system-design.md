# 战斗状态机设计文档

## 核心理念
战斗由代码状态机严格控制，DM 只负责叙事渲染。

## 状态流转

```
探索模式 → EnterCombat工具 → 战斗初始化 → 玩家回合 ⇄ 怪物回合 → 战斗结束 → 探索模式
```

## 关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 战斗中自由输入 | 禁止，只能点按钮 | 防止 DM 解释歧义 |
| DM 战斗角色 | 每个动作后写 1-2 句描写 | 保留叙事感但不控制机制 |
| 目标选择 | 按钮点击 | 无歧义 |
| 战斗进入 | 只能通过 EnterCombat 工具 | 防止 DM 叙事开战 |
| 回合制 | 玩家→全体怪物→下一轮 | 匹配现有实现 |

## 前端战斗 UI
- 战斗头部：轮次 + "你的回合/怪物回合"
- 敌人面板：立绘 + HP 条（常驻）
- 操作按钮：攻击/法术/物品/逃跑/防御
- 目标选择器：多敌时弹出
- 法术选择器：显示剩余次数
- 隐藏文本输入框

## 实现分 4 阶段
1. 后端状态机（types + combat-manager + engine）
2. 前端战斗 UI
3. 防御动作 + 边界处理
4. 测试
