# 战棋系统设计文档

> 版本 0.1 — 设计阶段，尚未实现

## 1. 设计哲学

**一句话**：火焰纹章的操作手感，TRPG 的叙事灵魂。

**三条铁律**：
1. **简化 > 完备**：7×5 棋盘、3 种地形、曼哈顿距离。能用 1 个数字解决的不用 2 个。
2. **位置即决策**：每回合玩家的核心问题是"站哪"，不是"按什么按钮"。
3. **代码管棋盘，DM 管叙事**：格子、距离、伤害全由代码计算。DM 只描述"格雷格挡在你面前，蛛母的毒牙咬在了他的盾上"。

## 2. 棋盘规格

### 2.1 尺寸

**7 列 × 5 行 = 35 格**

```
列: 0   1   2   3   4   5   6
行:
 0  .   .   M   .   M   .   .    ← 怪物侧（北）
 1  .   █   .   ▒   .   █   .
 2  .   .   .   .   .   .   .    ← 中线
 3  .   ▒   .   .   .   ▒   .
 4  .   A   .   P   .   A   .    ← 玩家侧（南）
```

**为什么 7×5**：
- 最大曼哈顿距离 = 10，射程 4 的远程能覆盖大约一半棋盘 → 站位有意义
- 初始阵距 ≈ 4 行，移动 3 的单位需要 2 回合接敌 → 留出走位空间
- 移动 4 的快速单位 1 回合能接敌 → 速度差异可感知
- 35 格足够放 2-6 个怪 + 玩家 + 2 盟友，不拥挤

### 2.2 距离

**曼哈顿距离**：`dist(a, b) = |a.x - b.x| + |a.y - b.y|`

不用切比雪夫（对角线算 1），原因：
- 曼哈顿在小棋盘上让近战 vs 远程差距更明显
- 实现简单，玩家直觉好理解（"隔了几格"）
- 移动范围是菱形而非正方形，视觉更清晰

### 2.3 坐标系

- `x` = 列 (0-6, 从左到右)
- `y` = 行 (0-4, 从上到下)
- 玩家侧 = y=4（南），怪物侧 = y=0（北）

## 4. 回合流程

### 4.1 行动模型

**每回合选一项（三选一）**：

```
A. 移动 + 攻击（火纹式一体化）
   → 点击"攻击" → 高亮所有可达+可攻击的目标
   → 点击目标 → 角色自动移动到最优攻击位 → 攻击
   → 回合结束

B. 纯移动（不攻击）
   → 点击想去的格子 → 角色移动过去
   → 回合结束

C. 原地行动（不移动）
   → 施法 / 使用物品 / 防御 / 逃跑
   → 回合结束
```

**注意**：施法不附带移动。法师要先用 B 走位，下回合再施法。这让位置选择有代价。

### 4.2 移动 + 攻击的计算

```
输入：unit（位置、移动力、攻击射程）、棋盘状态
输出：可攻击目标列表 + 每个目标对应的最优攻击位置

1. BFS 计算所有可达格子（考虑障碍物、敌方阻挡、困难地形）
2. 对于每个可达格子，检查该格子攻击射程内的敌人
3. 合并去重 → 可攻击目标列表
4. 对于每个目标，选择距离目标最远的可达攻击位（远程优先保持距离）
   近战例外：选择距离目标最近的可达位（冲上去）
```

### 4.3 可达格子的 BFS

```typescript
function getReachable(grid: Grid, start: Pos, moveSpeed: number): Map<string, number> {
  // 返回 Map<"x,y" → cost>，cost ≤ moveSpeed 的所有可达格子
  const queue: Array<[Pos, number]> = [[start, 0]]
  const visited = new Map<string, number>()
  visited.set(key(start), 0)

  while (queue.length > 0) {
    const [pos, cost] = queue.shift()!
    for (const neighbor of getNeighbors(pos)) {
      if (isWall(grid, neighbor)) continue            // 障碍物
      if (hasEnemy(grid, neighbor)) continue           // 敌方阻挡
      const moveCost = isDifficult(grid, neighbor) ? 2 : 1
      const newCost = cost + moveCost
      if (newCost > moveSpeed) continue
      const k = key(neighbor)
      if (visited.has(k) && visited.get(k)! <= newCost) continue
      visited.set(k, newCost)
      queue.push([neighbor, newCost])
    }
  }
  return visited  // 包含起点（cost=0）
}
```

### 4.4 远程阻挡判定

远程攻击被障碍物阻挡（简化版，不做完整 LOS）：

```
规则：如果攻击者和目标之间的直线路径上有障碍物格子，攻击被阻挡。

简化判定：
  dx = target.x - attacker.x
  dy = target.y - attacker.y
  steps = max(|dx|, |dy|)
  逐步检查路径上的格子（Bresenham 或简单整数步进）
  任一格子是墙 → 被阻挡
```

**友方不阻挡远程**——可以越过队友射击。

## 6. 怪物 AI

### 6.1 AI 类型

**近战冲锋型**（大部分小怪）：
```
1. 找到权重最高的目标（沿用现有权重系统：player=2, 嘲讽=3, 其他=1）
2. 如果当前位置可以攻击目标 → 攻击
3. 否则 BFS 最短路径到目标相邻格 → 移动 → 如果到了就攻击
4. 如果移动力不够到达 → 走到最近可达点
```

**侧翼包抄型**（野狼、兽人）：
```
同上，但选目标相邻格时优先选"目标背后"（y 方向更大的格子）
如果背后有障碍或已占据 → 退化为普通冲锋
```

**远程保距型**（暗影编织者、艾琳娜 NPC）：
```
1. 如果当前位置 dist(target) ≤ attackRange 且 > 1 → 攻击（不移动）
2. 如果 dist(target) ≤ 1（被贴脸） → 尝试后退 1 格再攻击
3. 否则 → 移动到 dist(target) = attackRange 的位置
```

**Boss 专用**：

- **蛛母**：近战冲锋 + 召唤时小蜘蛛出生在蛛母周围空格
- **暗影编织者**：远程保距 + 分身出现在棋盘对角线空格
- **蚀日兽**：中距冲锋（射程 2，会逼近但不贴脸） + 虚空脉冲全屏无视距离

### 8.2 交互流程

**攻击流程**：
1. 点击 `[攻击]` 按钮
2. 棋盘上高亮所有可攻击目标（红色边框/闪烁）
3. 同时高亮移动范围（蓝色半透明）
4. 玩家点击一个红色目标
5. 角色自动滑动到最优攻击位 → 攻击动画 → 伤害数字
6. 回合结束，进入敌方阶段

**移动流程**：
1. 点击 `[移动]` 按钮（或直接点击空格子）
2. 蓝色高亮可达范围
3. 点击目标格
4. 角色滑动过去
5. 回合结束

**施法流程**：
1. 点击 `[法术]` → 展开法术列表
2. 选择一个法术 → 高亮射程内的合法目标
3. 点击目标 → 施法动画 → 效果
4. 回合结束（不移动）

### 8.5 动画

- **移动**：`CSS transition: transform 0.3s`，逐格滑动
- **攻击**：目标格闪红 0.2s + 伤害数字浮出
- **受击**：单位图标 shake 动画 0.3s
- **死亡**：淡出 + 缩小 0.5s
- **召唤**：从 Boss 格向外弹出 0.4s

### 9.2 前端本地计算（不走后端）

以下计算在前端完成以保证交互响应性：
- **可达格子高亮**：前端持有 grid 状态，本地 BFS
- **可攻击目标高亮**：前端本地计算 reachable + range check
- **远程阻挡判定**：前端本地计算

后端只做**验证和执行**：收到玩家的 `{ type: 'combat_move', target: pos }` 或 `{ type: 'combat_attack', targetId, moveToPos }` 后，验证合法性，执行伤害计算，返回结果事件。

## 11. 现有条件在棋盘上的效果

| 条件 | 棋盘效果 |
|------|---------|
| restrained（束缚） | moveSpeed 变为 0，不能移动，可以原地攻击 |
| paralyzed（麻痹） | moveSpeed 变为 0，不能行动 |
| grappled（擒拿） | moveSpeed 变为 0，不能移动 |
| frightened（恐惧） | 不能主动向恐惧源移动（只能远离或原地） |
| blinded（致盲） | 攻击劣势（现有逻辑），无棋盘额外效果 |

## 12. 不做的东西（明确排除）

- 机会攻击 / ZOC 控制区
- 高低差 / 地形高度
- 视线计算（LOS）— 只有远程阻挡的简化判定
- 手动布阵
- 武器三角
- 迷雾 / 战争迷雾
- 多层地图
- 单位朝向 / 背刺
- 大体型（2×2 格）— Boss 也占 1 格
