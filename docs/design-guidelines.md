# TRPG Agent 设计规范

> 从破晓镇开发中沉淀的经验。适用于所有基于 Agent SDK 的游戏项目。

---

## 一、核心原则

### 代码管规则，LLM 管叙事
- **确定性计算**（掷骰、伤害、AC、检定）必须由代码完成，不交给 LLM
- **叙事描写**（场景、NPC 对话、气氛）交给 LLM 自由发挥
- LLM 收到的是工具返回的**事实结果**，基于事实编织故事
- 这样规则 100% 一致，叙事 100% 生动

### 事实存代码，记忆靠注入
- 玩家属性、NPC 信任度、已完成任务等**关键事实**存在 GameFactStore 中
- 每轮注入 DM prompt，即使对话被压缩也不丢事实
- LLM 的"记忆"不可靠（长对话会遗忘），代码的数据永远准确

### 物理世界一致性
- NPC 有固定位置，玩家必须在同一地点才能交互
- 所有工具（Talk/Attack/UseItem.give）都要做位置检查
- DM 不能凭空让不在场的 NPC 出现
- 地图连接关系决定可达性

## 二、战斗平衡

### Lv1 玩家基准线
```
剑士: HP 12, 攻击 d20+5, 伤害 1d6, AC 11
法师: HP 8, 攻击 d20+5, 伤害 1d10, AC 11
游侠: HP 10, 攻击 d20+5, 伤害 1d6, AC 13
牧师: HP 10, 攻击 d20+3, 伤害 1d6, AC 10
```

### 怪物设计规则
| 等级段 | DC 范围 | HP 范围 | 单次伤害上限 | 说明 |
|--------|---------|---------|-------------|------|
| 入门（暮色森林边缘） | 11-12 | 7-13 | 玩家 HP 的 40% | 新手能赢 |
| 普通（森林深处/矿道上层） | 12-13 | 13-20 | 玩家 HP 的 50% | 有挑战但不绝望 |
| 精英（矿道中层/荒原） | 13-14 | 20-30 | 玩家 HP 的 60% | 需要策略和药水 |
| Boss（矿道深处） | 15-16 | 40+ | 玩家 HP 的 80% | 需要满级+装备 |

### 关键原则
- **不能一击秒杀 Lv1 玩家**（最高伤害 < 玩家 HP）
- 特殊能力是**调味品不是杀手**（束缚 1 轮、STR-1，不是 STR-1d4）
- 每个怪物要有**一个记忆点**（蜘蛛的网、拟态怪的伪装、幽灵的力量吸取）
- 入门怪必须让玩家**有胜利感**

## 三、NPC 交互设计

### 信任度系统
```
-5 ~ -3: 敌对（拒绝交流，可能攻击）
-2 ~ -1: 冷淡（只说必要的话）
    0:   陌生人（基本礼貌）
 1 ~  2: 熟人（愿意多聊几句）
 3 ~  4: 朋友（分享秘密和线索）
    5:   挚友（全力帮助）
```

### 信任度变化规则
- **说服/交涉成功**: +1
- **帮助 NPC 相关的人**: +1
- **兑现承诺**: +2
- **伤害 NPC 在意的人**: -2
- **说谎被发现**: -1
- **普通对话**: DM 标注 [信任变化:NPC:±N:原因]
- **每次变化不超过 ±2**

### 信息揭示层
- 每个 NPC 有预定义的信息层级，按信任度自动解锁
- **确定性控制**：不依赖 LLM 判断该不该透露
- 信息分类：性格 / 背景 / 关系 / 线索 / 秘密
- 秘密之间应该**互相关联**形成网状结构

### NPC 设计清单
每个有名有姓的 NPC 必须有：
- [ ] 外貌描述（具体到疤痕、习惯动作）
- [ ] 说话方式（口头禅、语速、什么时候沉默）
- [ ] 核心信念（什么不妥协）
- [ ] 恐惧（什么让他害怕）
- [ ] 至少一个秘密（与主线相关）
- [ ] 和其他 NPC 的具体关系（不是泛泛的"朋友"）
- [ ] ASCII 画像
- [ ] 档案揭示层（5-10 层）

## 四、玩家引导设计

### 渐进揭示三阶段
```
第一阶段（前 3 轮）: 硬编码引导
  → 代码层注入场景提示，不依赖 LLM
  → 确保玩家到达第一个 NPC

第二阶段（4-10 轮）: NPC 引导
  → 每个 NPC 首次见面给一个明确方向
  → 任务系统给出结构化目标

第三阶段（10+ 轮）: 自由探索
  → 玩家已了解世界，自主决定行动
  → 防卡机制兜底（2 轮无行动 → NPC 事件触发）
```

### 防卡机制
- 检测"空闲输入"（短于 4 字、"嗯/看看" 等）
- 连续 2 轮空闲 → 按当前位置触发 NPC 事件
- 每个事件只触发一次，不重复
- 事件是**引导不是干预**（"小莉跑来说有人找你" 而非 "系统强制你去公会"）

### 开场设计模板
1. **角色创建**：名字 + 职业选择（4 选 1 够了）
2. **说书人开场**：硬编码世界背景（不靠 LLM）
3. **DM 第一幕**：设定场景，引入第一个 NPC
4. **第一个 NPC 给方向**：告诉玩家下一步去哪

### 任务设计规则
- 目标描述必须包含**地点提示** [暮色森林]
- 目标措辞用**具体动作**而非模糊指令
  - ✅ "在猎人石屋找到老林并交谈"
  - ❌ "找到猎人老林"
- 任务是**引导线不是限制** — 玩家可以不做
- /quest 命令要显示**进度条和下一步建议**

## 五、安全设计

### 三级安全模型
```
pass:  正常游戏内容 → 继续
warn:  不当但可处理 → 注入 DM 指令（NPC 制止、卫兵出现）
block: 现实犯罪/有害 → 强制终止游戏
```

### 设计原则
- Block 列表用**正则匹配关键词**，不依赖 LLM 判断
- Warn 不是惩罚，是**游戏内后果**（攻击 NPC → NPC 反击）
- 任何时候都不模拟现实暴力/自伤场景

## 六、技术约定

### 工具设计
- 工具返回**事实文本**，不返回叙事
  - ✅ "攻击判定: d20(17)+3=20 vs DC12 → 命中。造成 5 点伤害。"
  - ❌ "你的剑精准地劈中了敌人..."（这是 DM 的工作）
- 每个工具必须声明 `isConcurrencySafe` 和 `isReadOnly`
- 写操作工具必须做**位置检查**

### 存档设计
- GameSession 完整 JSON 序列化
- 多存档支持（quicksave / autosave / 自定义名）
- 每 5 轮自动存档
- 退出时自动存档

### DM Prompt 设计
- 分为：角色定义 + 世界摘要 + NPC 快速参考 + 工具指南 + 引导规则
- **每轮注入 GameFactStore 上下文**（权威事实来源）
- 明确告诉 DM："游戏状态摘要优先于你的记忆"
- 位置规则、信息节奏、防卡机制都写进 prompt

## 七、开发流程

### 设计先行
1. 先写设计文档（world.md, characters.md, mechanics.md, player-journey.md）
2. Team review 设计文档
3. 再写代码实现

### 分工原则
- 数据层（types + GameFactStore）先做
- 规则层（RulesEngine + tools）第二
- DM 层（prompt + agent）最后
- 不同层改不同文件，可以并行

### Review 节奏
- 每个版本完成后做一轮 alignment review（设计 vs 代码）
- 记录发现的新问题，不在当前版本修
- 下个版本集中处理 follow-up
- 每轮 review 后才能发布
