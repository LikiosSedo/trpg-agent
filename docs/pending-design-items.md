# 待设计/实现项目清单

## 优先级 1：底层机制

### 1. 物品基础类型模板
动态创建的物品需要自动映射到基础模板获得机械效果。
- potion 类 → 默认 bonus=2 (恢复2d4+2)
- weapon 类 → 默认 bonus=0 (1d6)
- 详见 docs/item-types-and-trade-design.md

### 2. 交易砍价闭环优化
当前：DM 调 ProposeTradeAction 弹卡片，砍价按钮关闭卡片回到对话。
待优化：砍价后 DM 重新报价是否可靠？需要测试验证。

### 3. NPC 主动攻击的叙事过渡
暴力后果系统已实现 2 阶段响应（到达 → 下轮战斗），但 DM 叙事过渡质量取决于上下文注入。需要测试验证。

## 优先级 2：体验优化

### 4. 物品名称中英文统一
背包显示混合中英文（Healing Potion vs 治疗药水）。需要统一显示名。

### 5. 战斗叙事增强
战斗目前是纯数据日志。可以让 DM 在战斗间隙添加 1-2 句叙事描写。

### 6. 章节推进可视化
章节推进时应该有明显的过场提示（类似章节标题卡）。

## 优先级 3：长期规划

### 7. 多人支持
全局模块状态需要隔离（per-session scoping）。

### 8. NPC 日常作息
不同时间 NPC 自动移动到不同位置（白天在岗位，晚上回家）。

### 9. 存档管理 UI
多存档支持、存档命名、删除确认。
