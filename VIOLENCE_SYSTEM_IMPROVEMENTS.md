# 暴力系统和战斗叙事改进计划

## 概述

本文档规划了暴力系统的全面改进，包括全镇信任度动态传播、NPC 跨区域追踪、战斗叙事闭环等功能。

---

## 需求分析

### 1. 暴力系统触发时全镇信任度动态下降
**当前问题**：
- 暴力事件触发后，只有直接响应者（responder）的信任度会变化
- 其他 NPC 不会感知到暴力事件，信任度保持不变
- 无法形成"全镇公敌"的效果

**目标**：
- **触发时机**：不是暴力发生时，而是**延迟轮次结束、响应者到达现场后**才触发信任度传播
  - 逻辑：暴力事件被"公开化"了，消息传开了
  - 场景 1：响应者到达 → 战斗发生 → 战斗结束后触发信任度传播
  - 场景 2：响应者到达但玩家已逃离 → 响应者发现现场 → 触发信任度传播
  - 场景 3：玩家击败响应者 → 更多人知道了 → 触发信任度传播
- **下降幅度有区别**：
  - 受害者的 bond NPC（亲近关系）：-8 到 -10（直接进入敌对）
  - 响应者的 bond NPC（连坐）：-5 到 -7
  - 目击者（当时在场的 NPC）：-5 到 -7
  - 其他镇民：-3 到 -5（听说消息）
- **信任度下降后，NPC 的行为应该自动改变**：
  - 战斗型 NPC（canFight=true）：信任度 ≤ combat 阈值时自动触发战斗响应
  - 非战斗型 NPC：拒绝对话、拒绝交易、逃避玩家

### 2. DM 提示词优化
**当前问题**：
- DM 提示词中关于信任度和行为的指导不够明确
- 缺少对暴力后果的叙事指导
- 缺少对 NPC 行为变化的明确说明

**目标**：
- 在 DM 提示词中明确说明：信任度不仅影响信息透露和态度，还影响行为
- 添加暴力后果的叙事指导：
  - 全镇恐慌氛围
  - NPC 的回避、敌对、报复行为
  - 不同信任度等级的 NPC 反应差异
- 强调 DM 的参与：根据信任度主动调整 NPC 行为和对话

### 3. NPC 跨区域追踪机制
**当前问题**：
- 玩家逃离暴力现场（如从镇上逃到森林），NPC 不会追踪
- 只有在同一 location 才会触发战斗
- 战斗能力强的 NPC（如艾琳娜、韩猛）应该能够追踪玩家

**目标**：
- 实现 NPC 跨区域追踪机制：
  - 战斗能力强的 NPC（canFight=true + 特定标记）可以追踪玩家
  - 追踪范围：镇上 ↔ 森林、镇上 ↔ 矿道入口
  - 追踪延迟：2-3 轮后到达
  - 追踪叙事：DM 描述 NPC 追踪过程（脚印、气息、询问目击者）
- 追踪触发条件：
  - 玩家在暴力事件后逃离现场
  - 响应者有追踪能力（新增 canTrack 标记）

### 4. 战斗前后叙事闭环
**当前问题**：
- 战斗前叙事不完整：
  - 遭遇战缺少环境铺垫
  - NPC 主动攻击缺少动机说明
- 战斗后叙事缺失：
  - 战斗结束后直接回到探索模式
  - 缺少战斗结果的叙事描写（敌人倒地、逃跑、投降）
  - 缺少环境变化描写（血迹、破坏、尸体）
- 战斗后没有生成选项：
  - 玩家不知道下一步该做什么
  - 缺少 beat 推进选项
  - 缺少 DM 引导

**目标**：
- **战斗前叙事**：
  - 遭遇战：DM 描写环境线索（脚印、声音、气味）→ 怪物出现 → 战斗开始
  - NPC 主动攻击：DM 描写 NPC 到达 → 质问/愤怒 → 战斗开始
  - 玩家主动攻击：DM 描写攻击动作 → 战斗开始
- **战斗后叙事**：
  - 战斗结果描写：
    - 胜利：敌人倒地/逃跑/投降，环境变化
    - 失败：玩家倒地，Game Over 或昏迷
  - 战利品描写：DM 描述从敌人身上搜到什么
  - 环境恢复：DM 描述战斗后的场景（安静、血迹、破坏）
- **战斗后生成选项**：
  - 自动调用 SetActions 生成选项：
    - 搜索尸体/战利品
    - 离开现场
    - 休息恢复
    - Beat 推进选项（如果有相关 beat）
    - DM 引导选项（下一步建议）

### 5. 战斗状态机闭环
**当前问题**：
- 战斗流程不是完整的状态机
- 各个环节之间缺少明确的状态转换
- 容易出现状态不一致的 bug

**目标**：
- 设计完整的战斗状态机：
  ```
  [探索模式]
      ↓ (触发战斗)
  [战斗前叙事] ← DM 描写环境/动机
      ↓
  [战斗初始化] ← 系统生成 combat_init 事件
      ↓
  [战斗回合] ← 玩家行动 + 怪物行动
      ↓ (战斗结束)
  [战斗后叙事] ← DM 描写结果/环境
      ↓
  [生成选项] ← SetActions 生成下一步选项
      ↓
  [探索模式]
  ```
- 每个状态都有明确的：
  - 入口条件
  - 处理逻辑
  - 出口条件
  - 事件生成
- 防止状态跳跃和遗漏

## 实现计划

### 阶段 1：全镇信任度传播机制

**任务 1.1：设计信任度传播规则**
- 文件：`src/trust-system.ts`
- 新增函数：`propagateViolenceTrust(session, victimName, responderName, witnessNames)`
- 逻辑：
  1. 识别受害者的 bond NPC（从 npc-relationships.ts 读取）
  2. 识别响应者的 bond NPC（连坐效应）
  3. 识别目击者（暴力发生时在场的 NPC，从 violence_alert 记录中读取）
  4. 识别其他镇民（所有其他 NPC）
  5. 分别应用信任度变化：
     - 受害者 bond NPC: -8 到 -10
     - 响应者 bond NPC: -5 到 -7
     - 目击者: -5 到 -7
     - 其他镇民: -3 到 -5
  6. 记录变化日志

**任务 1.2：在 violence_alert 中记录目击者**
- 文件：`src/types.ts`
- 修改 `violenceAlerts` 类型，添加 `witnesses: string[]` 字段
- 文件：`src/engine.ts`
- 修改创建 violence_alert 的代码，记录当时在场的所有 NPC（除了受害者）

**任务 1.3：集成到暴力系统**
- 文件：`src/engine.ts`
- 修改位置：violence_alert 响应逻辑（阶段 2：战斗结束或玩家逃离后）
- 调用时机：
  - 场景 1：`alert.responded = true` 时（响应者到达且战斗即将开始）
  - 场景 2：玩家逃离现场时（`session.worldState.currentLocation !== alert.location`）
  - 场景 3：战斗结束后（combat.ended）
- 调用：`propagateViolenceTrust(session, alert.victimName, alert.arrivedResponder, alert.witnesses)`
- 生成事件：`trust_cascade` 事件，显示信任度传播结果

**任务 1.4：DM 手动触发信任度传播工具**
- 文件：`src/tools/trigger-trust-cascade.ts`
- 新增工具：`TriggerTrustCascade`
- 功能：允许 DM 在意外情况下手动触发信任度传播
- 参数：
  - `victimName: string` - 受害者名称
  - `reason: string` - 触发原因（用于日志和叙事）
  - `witnesses?: string[]` - 可选的目击者列表（如果不提供，使用当前在场的所有 NPC）
- 使用场景：
  - DM 叙事中玩家的暴行被意外发现（如尸体被发现、血迹被看到）
  - 玩家在子区域内攻击 NPC，其他子区域的 NPC 听到声音赶来
  - 特殊剧情需要立即触发全镇敌对
- 调用后：
  1. 立即调用 `propagateViolenceTrust`
  2. 生成 `trust_cascade` 事件
  3. 返回信任度变化摘要给 DM

**任务 1.5：注册工具到 DM Agent**
- 文件：`src/dm-agent.ts`
- 将 `TriggerTrustCascade` 添加到 DM 的工具列表
- 文件：`src/dm-prompt.ts`
- 在工具使用指南中添加说明：
  ```markdown
  | TriggerTrustCascade | 当暴力事件被意外发现或需要立即触发全镇敌对时使用。通常由暴力系统自动处理，但在特殊剧情需要时可手动调用。 |
  ```

**任务 1.6：前端显示信任度变化**
- 文件：`public/index.html`
- 新增事件处理：`trust_cascade` 事件
- 显示格式：
  ```
  ⚠️ 暴力事件震惊全镇！
  - 小莉（亲近关系）：信任度 -10
  - 陈妈（目击者）：信任度 -6
  - 其他镇民：信任度 -4
  ```

### 阶段 2：DM 提示词优化

**任务 2.1：审查现有提示词**
- 文件：`src/dm-prompt.ts`
- 审查内容：
  - 信任度相关的说明是否完整
  - 暴力后果的叙事指导是否清晰
  - NPC 行为变化的说明是否明确

**任务 2.2：添加信任度行为指导**
- 在 DM 提示词中添加章节：
  ```markdown
  ## 信任度与 NPC 行为
  
  信任度不仅影响 NPC 的态度和信息透露，还直接影响 NPC 的行为：
  
  ### 信任度等级与行为
  - **友好（trust ≥ 5）**：主动帮助、提供折扣、透露秘密
  - **中立（0 ≤ trust < 5）**：正常交易、基本对话
  - **冷淡（-5 ≤ trust < 0）**：简短回答、拒绝帮助
  - **敌对（-8 ≤ trust < -5）**：拒绝对话、拒绝交易、回避玩家
  - **极度敌对（trust < -8）**：
    - 战斗型 NPC：主动攻击或召唤守卫
    - 非战斗型 NPC：逃跑、躲藏、报警
  
  ### 暴力后果叙事
  当玩家对 NPC 施暴后，全镇信任度会动态下降。你需要在叙事中体现：
  - 镇民的恐慌和愤怒
  - NPC 的回避行为（看到玩家就躲开）
  - 商人拒绝交易（"我不想和你这种人做生意"）
  - 守卫的警惕（"你最好离开这里"）
  - 亲近关系 NPC 的极度愤怒（"你怎么能这么做！"）
  ```

**任务 2.3：添加战斗叙事指导**
- 在 DM 提示词中添加章节：
  ```markdown
  ## 战斗叙事闭环
  
  战斗不是孤立的数值计算，而是叙事的一部分。你需要在战斗前后提供完整的叙事：
  
  ### 战斗前叙事
  - **遭遇战**：描写环境线索（脚印、声音、气味）→ 怪物出现的瞬间
  - **NPC 主动攻击**：描写 NPC 到达现场 → 愤怒/质问 → 拔剑/施法
  - **玩家主动攻击**：描写玩家的攻击动作 → 目标的反应
  
  ### 战斗后叙事
  战斗结束后，系统会注入 [战斗结束] 标记。你需要：
  1. 描写战斗结果（敌人倒地/逃跑/投降）
  2. 描写环境变化（血迹、破坏、尸体）
  3. 描写玩家状态（疲惫、受伤、喘息）
  4. 如果有战利品，描写搜索过程
  5. 最后调用 SetActions 生成下一步选项
  ```

**任务 3.1：设计追踪能力标记**
- 文件：`src/data/npc-personalities.ts`
- 新增字段：`canTrack: boolean`
- 标记追踪能力强的 NPC：
  - 艾琳娜（精灵游侠，追踪专家）
  - 韩猛（经验丰富的战士）
  - 格雷格（前佣兵，有追踪经验）

**任务 3.2：实现追踪逻辑**
- 文件：`src/engine.ts`
- 新增函数：`checkCrossRegionTracking(session)`
- 逻辑：
  1. 检查是否有未响应的 violence_alert

     - 目标位置：玩家当前位置
     - 追踪叙事：DM 描写追踪过程

  - {NPC名}询问目击者或追踪痕迹

### 阶段 5：战斗状态机重构

**任务 5.1：定义战斗状态枚举**
- 文件：`src/types.ts`
- 新增类型：
  ```typescript
  export type CombatPhase = 
    | 'none'              // 非战斗状态
    | 'pre_narrative'     // 战斗前叙事
    | 'initializing'      // 战斗初始化
    | 'in_progress'       // 战斗进行中
    | 'post_narrative'    // 战斗后叙事
    | 'generating_options' // 生成选项
  
  export interface CombatState {
    active: boolean
    phase: CombatPhase
    // ... 其他字段
  }
  ```

**任务 5.2：实现状态转换逻辑**
- 文件：`src/combat-manager.ts`
- 新增函数：`transitionCombatPhase(session, newPhase)`
- 验证状态转换的合法性
- 记录状态转换日志

**任务 5.3：重构战斗触发流程**
- 确保所有战斗触发都遵循状态机：
  1. `none` → `pre_narrative`（注入战斗前叙事标记）
  2. `pre_narrative` → `initializing`（DM 完成叙事后）
  3. `initializing` → `in_progress`（生成 combat_init 事件）
  4. `in_progress` → `post_narrative`（战斗结束）
  5. `post_narrative` → `generating_options`（DM 完成叙事后）
  6. `generating_options` → `none`（SetActions 调用后）

## 优先级排序

1. **高优先级**（核心功能，影响游戏体验）：
   - 阶段 1：全镇信任度传播机制
   - 阶段 4：战斗叙事闭环

2. **中优先级**（增强体验，但不影响核心玩法）：
   - 阶段 2：DM 提示词优化
   - 阶段 3：NPC 跨区域追踪机制

3. **低优先级**（代码质量改进，防止未来 bug）：
   - 阶段 5：战斗状态机重构
